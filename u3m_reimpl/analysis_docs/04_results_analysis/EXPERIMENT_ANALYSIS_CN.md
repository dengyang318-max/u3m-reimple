# 实验结果异常分析：Tail 区域隐含信息识别（四组实验综合分析）

## 1. 分析框架

根据论文描述，通过 Ray Sweeping 方法找到的 tail 区域应该展现以下**异常模式**，以表明发现了隐含信息：

1. **标签分布异常**：Tail 中的正样本率与全局显著不同
2. **模型性能异常**：Tail 上的 Accuracy/F1 与全局显著不同（可能更好或更差）
3. **多 Percentile 趋势**：随着 percentile 降低，指标呈现明显的变化趋势

## 2. 实验概述

本报告分析四组实验结果，识别发现的少数群体特征，并进行横向对比：

1. **Chicago Crimes (20251204_223121)**: Original vs Updated 方法对比
2. **Chicago Crimes Official Utils (20251204_220522)**: 官方工具方法
3. **College Admission (20251204_201425)**: Original vs Updated 方法对比
4. **College Admission Official Utils (20251205_144132)**: 官方工具方法

所有实验都使用了：
- 全部数据（不进行采样）
- 多 Percentile 评估：`[1.0, 0.1, 0.01, 0.001, 0.0001]`（Chicago Crimes）或 `[1.0, 0.5, 0.2, 0.1, 0.08, 0.04]`（College Admission）
- 相同的模型：LogisticRegression（在全部数据上训练）

**发现的真实隐含信息**：

   - 发现了"高录取率区域"的隐含模式
   - Admit Rate 提升（+0.118 到 +0.157），F1 显著提升（+0.209 到 +0.305）
   - 说明某些 GRE/GPA 组合的极端区域，录取率更高，且模型在这些区域上的 F1 提升

2. ✅ **Updated 和 Official Utils 方法的 Top-2/Top-3 方向**：
   - 也发现了"高录取率区域"的隐含模式
   - 与 Original 方法发现的模式一致

   - F1=0 的严重异常
   - **这本身就是重要的隐含信息**: 说明这些方向捕获了模型训练时未充分覆盖的区域

**方法差异分析**：

1. **Original vs Updated 方法**：
   - Original 方法在 Top-1 方向发现了显著异常（F1 提升 +0.209）
   - Updated 方法在 Top-1 方向发现了模型失效（F1=0）
   - **可能原因**: min-shift 和 vector_transfer 的改进改变了算法捕获的方向，发现了不同的隐含信息

2. **Updated vs Official Utils 方法**：
   - 两者都发现了模型失效（F1=0），但出现在不同的方向
   - **可能原因**: 实现细节的差异（数据预处理、模型配置等）导致捕获了不同的隐含信息

3. **结论**：
   - ✅ **不同方法发现的隐含信息确实不同**，这不一定是因为实现错误
   - ✅ **可能是算法本身的特点**: 不同的预处理、不同的方向选择策略，会捕获不同的隐含信息
   - ✅ **模型失效（F1=0）本身就是重要的发现**: 说明这些 tail 区域是模型训练时未充分覆盖的"盲区"
